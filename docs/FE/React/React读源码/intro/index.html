源码概述#

External Dependencies: React has almost no external dependencies.The fbjs repository (Facebook projects)exists because React shares some small utilities with libraries like Relay.

Top-Level Folders:

  • packages/

    • src/ If your change is related to the code, the src subdirectory of each package is where you’ll spend most of your time.

  • fixtures/: contains a few small React test applications for contributors.

  • build/: is the build output of React.

The React documentation website git repo There are a few other top-level folders but they are mostly used for the tooling and you likely won’t ever encounter them when contributing.

packages/react

Renderers#

packages/

  • React DOM Renderer enders React components to the DOM.-react-dom
  • React Native Renderer renders React components to native views- react-natives-renderer
  • React Test Renderer renders React components to JSON trees

Reconcilers#

packages/react-reconciler 视图层和渲染层的协调器 they are exclusively used by renderers such as React DOM and React Native.

By the end of the reconciliation, React knows the result DOM tree, and a renderer like react-dom or react-native applies the minimal set of changes necessary to update the DOM nodes (or the platform-specific views in case of React Native). 计算出virtualDOM，并diff，将需要渲染的提供给Renderers

  • Stack Reconciler
  • Fiber Reconciler

Event System#

packages/legacy-events React implements a synthetic event system which is agnostic of the renderers and works both with React DOM and React Native.

源码实现概述#

This section is a collection of implementation notes for the stack reconciler.

前置阅读：

  • 源码概述 | the codebase overview
  • React组件、实例和React元素的区别 | differences between React components, their instances, and elements 推荐阅读，对于深入理解React很有帮助

The stack reconciler was used in React 15 and earlier. It is located at src/renderers/shared/stack/reconciler.

The reconciler itself doesn’t have a public API. Renderers like React DOM and React Native use it to efficiently update the user interface according to the React components written by the user.

reconciler协调器没有公共的API,它只在底层提供给渲染器renderer（react-dom和react-native-renderer）
