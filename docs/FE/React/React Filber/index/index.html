Fiber简介#

React Fiber是React core核心算法 重要特性：incremental rendering增量渲染（将render工作切分为chunk并分发到多个帧中进行）

理解Fiber#

Reconciliation#

当React App渲染时，会生成一个描述当前应用的tree并保存在内存中。在浏览器渲染时，则将当前的vitual dom tree映射(or translate)到浏览器DOM中。当使用setState等API进行更新时，又重新生成一个新的new tree,从而diff计算new tree和上一次tree的差异，将差异的部分重新渲染。

尽管Fiber是这部分算法的重写，但是整体的思路是一致了，提高了性能。

diff算法中：

  • 组件类型的变化会生成新的子树，react不会去diff，而是直接替换旧的子树
  • 对于数组的diff是使用keys,Keys需要是稳定、可预测和唯一的

React之所以能够write once, run anywhere就是因为reconciliation和rendering是分离的（理论上来说，开发者只需关心reconciliation）

Scheduling#

计算、渲染等工作应该怎么按计划执行

这是fiber改进的地方

what is fiber#

A primary goal of Fiber is to enable React to take advantage of scheduling

Fiber做了什么？ 将各种工作分成小块,React的工作被切分为chunk并分发到多个帧中进行。

浏览器渲染UI时，如果js做了太多复杂的工作就会阻塞渲染，导致页面看起来像是掉帧了一样。而有些工作是不必要，可以被推迟。

Fiber为React组件重新实现了一套调用堆栈,以实现某些工作的分批处理，避免阻塞UI渲染。这个堆栈可以称为virtual stack frame虚拟栈帧
