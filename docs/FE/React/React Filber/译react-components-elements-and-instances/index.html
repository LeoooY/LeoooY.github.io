React 组件（Components）, 元素（Elements）和实例（Instances）#

2015年12月18日 Dan Abramov 译：2020年6月11日 Leo Yang

组件（Components）, 元素（Elements）和实例（Instances）之间的区别常常使很多React初学者很困惑。为什么屏幕上绘制的东西拥有三种不同的术语？

管理各个实例 Managing the Instances#

如果你才接触React，你可能只接触过class组件和instances实例。举个栗子，你可能会通过创建一个类来声明一个 Button 组件。当app运行的时候，你的屏幕上可能会有好几个 Button 的实例，每个实例都有其各自的properties和局部state。这就是传统的面向对象UI编程。为什么要引入elements的概念呢？

在传统的UI模型中，创建和销毁组件实例的操作需要你自行去处理。如果一个Form组件想渲染一个Button组件，需要创建它的实例，并且手动与新的状态保持同步。

class Form extends TraditionalObjectOrientedView {
  render() {
    // Read some data passed to the view
    const { isSubmitted, buttonText } = this.attrs;
    if (!isSubmitted && !this.button) {
      // Form is not yet submitted. Create the button!
      this.button = new Button({
        children: buttonText,
        color: 'blue'
      });
      this.el.appendChild(this.button.el);
    }
    if (this.button) {
      // The button is visible. Update its text!
      this.button.attrs.children = buttonText;
      this.button.render();
    }
    if (isSubmitted && this.button) {
      // Form was submitted. Destroy the button!
      this.el.removeChild(this.button.el);
      this.button.destroy();
    }
    if (isSubmitted && !this.message) {
      // Form was submitted. Show the success message!
      this.message = new Message({ text: 'Success!' });
      this.el.appendChild(this.message.el);
    }
  }
}

上面是伪代码，不过差不多就是当你用Backbone这样的库来编写以面向对象的方式运行的复合UI代码的结果。

每个组件实例需要保存它到DOM node和它的子组件实例的引用，并在相应的时候，创建、更新和销毁其子组件。代码量以状态和组件的平方量级增长，并且父子组件之间的耦合使得以后很难进行组件分离。

所以React有什么不同呢？

用元素来描述树 Elements Describe the Tree#

在React中，elements发挥作用。一个element是一个描述组件实例或者DOM node及其期望属性的普通的对象。它只包含组件类型（比如一个Button）,其属性（比如color）和它的所有子元素。

一个元素不是一个真正的实例。确切的说，它是一种方式来告诉React你想在屏幕上渲染什么。你不能调用这个对象的任何方法。它只是一个有两种属性的不变的描述：type: (string | ReactClass) 和 props: Object

DOM 元素 DOM Elements#

当一个元素的type属性是String类型的，那它就代表这个DOM节点的Tag名，prop则代表其属性。这就是React会去渲染的东西，举个栗子：

{
  type: 'button',
  props: {
    className: 'button button-blue',
    children: {
      type: 'b',
      props: {
        children: 'OK!'
      }
    }
  }
}

这个元素就是一种将下面的HTML表示成普通对象的形式：

<button class='button button-blue'>
  <b>
    OK!
  </b>
</button>

注意一下元素是如何嵌套的。按照惯例，当我们想创建一个元素树时，我们会指定一个或者多个子元素作为容器元素的children。

重要的是子元素和父元素都只是描述而不是真正的实例。当你创建它们时，它们的对象不会指向屏幕上的任何东西。你可以随意创建和销毁它们，这都不会有什么影响。

React元素的遍历很轻松，不需要做额外的解析，因为它们只是普通对象。当然就比真正的DOM元素更加轻量级。

组件元素 Component Elements#

元素的type也可以是React的function组件或class组件。

{
  type: Button,
  props: {
    color: 'blue',
    children: 'OK!'
  }
}

这是React的核心思想

描述组件的元素也是元素，就像描述DOM节点的元素一样。它们可以嵌套并相互混合。

通过此功能，您可以将DangerButton组件定义为具有特定color属性值的按钮，而无需担心按钮是否呈现为DOM<Button>、<div>或者其他内容：

const DangerButton = ({ children }) => ({
  type: Button,
  props: {
    color: 'red',
    children: children
  }
});

可以在单个元素树中混合搭配使用DOM和组件元素：

const DeleteAccount = () => ({
  type: 'div',
  props: {
    children: [{
      type: 'p',
      props: {
        children: 'Are you sure?'
      }
    }, {
      type: DangerButton,
      props: {
        children: 'Yep'
      }
    }, {
      type: Button,
      props: {
        color: 'blue',
        children: 'Cancel'
      }
   }]
});

或者使用JSX：

const DeleteAccount = () => (
  <div>
    <p>Are you sure?</p>
    <DangerButton>Yep</DangerButton>
    <Button color='blue'>Cancel</Button>
  </div>
);

这种混合搭配使用有助于保持组件之间的解耦，因为它们可以通过组合来表达is-a和has-a关系：

  • Button是一个具有特定属性的DOM<Button>
  • DangerButton是具有特定属性的按钮
  • DeleteAccount在<div>中包含一个Button和一个DangerButton
