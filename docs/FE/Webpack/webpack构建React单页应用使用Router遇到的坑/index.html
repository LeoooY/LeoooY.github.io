webpack构建React单页应用使用Router遇到的坑

前言#

最近尝试用webpack从零开始构建一个react单页项目,使用router进行路由切换时遇到踩了一些坑.

发现问题#

使用路由去访问应用时,报can not get xxxx 配置devServer的historyApiFallback为true即可.

devServer: {      
      historyApiFallback: true, 
}

当页面增加\路由复杂时,新的问题出现了. 我只有在页面中点击进行的跳转能够正常匹配到路由的路径,而不能直接从地址栏中跳转到配置的路径. what???

最开始,我将路由配置为 path="/x"的只有一个层级的路径,如下:

<Route path='/x' component={xxxxx}></Route>  
<Route path='/y' component={xxxxx}></Route>  
<Route path='/z' component={xxxxx}></Route>

此时,路由的表现是正常的.

但当我配置了几个多层级的路由: path="/x/x",

<Route path='/x/0' component={xxxxx}></Route> 
<Route path='/x/1' component={xxxxx}></Route> 
<Route path='/y/0' component={xxxxx}></Route>

路由就抽风了.

错误提示为

大概意思是: 找不到工程的入口文件.

我的第一反应是: 难道是我的router没配置对?(看来我对react router也不太熟悉...) 然而在反复对比官网示例后,我确信自己的router配置是正确的.

接着根据这个找不到工程的入口文件,我怀疑是htmlwebpackplugin output devServer配置不正确.

搜寻许久也没有什么卵用,我决定重新分析一下问题的表现.

首先,在根目录下/ /xx去访问都没有问题,并且路由在进去后,页面内触发的跳转也没有问题. 问题出现在直接使用链接去跳转到指定路径, 如使用/xx/xx试图进入页面就报错,找不到入口文件.

再看下报错信息,我去!!! 问题已经很明显了. 入口文件在/目录下,当然在/xx/目录下找不到了...

最终我用create-react-app生成了一个react应用, 然后yarn eject在其webpack.config的output中找到了一个神奇的配置项解决了我的问题!!!

devtoolModuleFilenameTemplate: info =>
path.resolve(info.absoluteResourcePath).replace(/\\/g, '/'),

Point sourcemap entries to original disk location (format as URL on Windows)
