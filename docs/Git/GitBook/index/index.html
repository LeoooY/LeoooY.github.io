1.2 A Short History of Git#

Linux之父linus在开发Linux kernal时候一开始使用一个分布式的版本管理系统BitKeeper。在2005年的时候，开发BitKeeper的公司破产了，BitKeeper不再免费使用。 于是Linus和社区的开发者就基于BitKeeper的使用经验等自己开发了一款分布式版本管理系统，这便是Git。

开发时，对Git的目标是：

  • Speed
  • Simple designed
  • Strong support for non-linear development(brantches)
  • Able to handle large projects like the Linux kernal efficientlty(Speed and Data Size) 自2005年开始，Git就遍地开花，现在已经非常成熟，使用简单且可靠。

    It’s amazingly fast, it’s very efficient with large projects, and it has an incredible branching system for non-linear development

1.3 Getting Start - What is Git?#

  • snapshot Git记录的是文件快照而不是文件的差异
  • local Operations 本地操作
  • Integrity 所有操作都被记录
  • Generally Only Adds Data to the Git database 数据丢失的概率非常小
  • The Three States 三颗树
    • Modified
    • Staged
    • Committed

basic Git workflow：

1 You modify files in your working tree.
2 You selectively stage just those changes you want to be part of your next commit, which adds only those changes to the staging area.
# git add -A / fname.xx
3 You do a commit, which takes the files as they are in the staging area and stores that snapshot permanently to your Git directory.
# git commit -m "xxdesc"

2.2 Git Basics - Recording Changes to the Repository#

git status [-s]                    # check the status of your file 
,gitignore                         # ignore files
git diff
git add [-A]
git commit -a -m 'added new benchmarks # commit with already tracked files
git rm <filename>              # Removing Files
git mv file_from file_to        # Moving Files
git checkout --track origin/daves_branch    # 拉取remote分支
git branch -d | -D branch_name              # 删除某个分支
git clone --depth=1  [repourl]              # clone库的最新revision，当某个库很大的时候适用

cherry-pick 从某个分支取部分提交合并到另一个分支：

a-b-c           # branch0
  \
   x-y-z        # branch1                
                # a、b、c、x、y、z is commit id
git checkout branch0
git cherry-pick y..z
# result
a-b-c-y-z
  \
   x-y-z

git rebase vs git merge

A---B---C topic
         /
    D---E---F---G master
 
 # 当前处于topic分支
 git rebase master      # 两条命令等价
 git rebase master topic
 # 操作结果：
                   A'--B'--C' topic
                 /
    D---E---F---G master
