安装 win10 + ubuntu18.04双系统#

  • 制作uefi模式的U盘启动盘 这一步非常关键。第一装做成了legacy模式的u盘启动盘，虽然改到lagacy模式后将ubuntu装上了，可是启动时候十分尴尬，只能从uefi模式可以进入win，easyBCD又不支持uefi模式。直接导致了装了之后要么开不起，要么只能用uefi进win...相当于白费。 用UtralISO实测，基本默认设置就ok.

  • win10系统盘中分出至少200m空间作为ubuntu的引导 具体操作：磁盘管理 > 找到系统盘 > 右键压缩磁盘空间 200可以根据空闲空间调整，我分了1.2G

  • 禁用win10快速启动

  • 禁用BIOS secure boot

  • 修改SATA模式为AHCI 不改为AHCI的话ubuntu安装时候无法识别win的分区

  • 插入U盘，BIOS设为U盘启动 由于第一步制作的是UEFI模式的启动盘，所以应该可以直接识别到

  • 安装ubuntu 该步骤主要注意分区的步骤，选择自行分区： 创建分区：

位置分区类型空间文件类型描述
/boot逻辑分区1.2Gext4用于启动引导，使用win系统盘分出来的1.2G
/主分区120Gext4用于启动引导，使用win系统盘分出来的1.2G
/swap逻辑分区30Gswap用于内存交换空间，由于内存有24G,所以设的比较大
/home逻辑分区320Gext4剩余的空间全部分配给/home

注：空间位置全部设置为空间起始位置 其他步骤应该一路next就好了。安装完成后选择重启，应该就可以看到引导中有win和ubuntu的选项了

  • 设置win通过AHCI启动 由于之前将SATA设置为AHCI，此时ubuntu没问题，win应该启动不了... 按照此步骤走一遍ubuntu和win就都正常了： install dual boot ubuntu with windows 10 and RAID on

无线网卡无法使用#

原因: ubuntu block了网卡硬件？？

rskill list all
# 可以看到一项ideapad_wlan处于
# sofe block: no
# hard block: yes
# 的状态

解决:

sudo vim /etc/modprobe.d/blacklist.conf
# 添加一项
blacklist ideapad_laptop
# 重启

sudo vim /etc/privoxy/config
# 1336行 forward-socks5t / 127.0.0.1:1080
service privoxy status|start|restart
# privoxy的默认妆发端口为8118
export http_proxy='http://localhost:8118'
export https_proxy='https://localhost:8118'
