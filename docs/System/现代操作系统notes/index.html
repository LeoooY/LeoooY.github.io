现代操作系统notes

作者: Andrew Ｓ. Tanenbaum 麻省理工学院计算机科学教授 ACM会士 IEEE会士 主研方向：系统安全

编写了MINIX,一个小型的UNIX www.minix3.org

第一章 引论#

现代操作系统由一个或多个处理器、主存、磁盘、打印机、键盘、鼠标、显示器、网络接口以及其他各种输入输出设备组成。 一般而言，现代操作系统是一个复杂的系统，如果每个应用程序员都不得不掌握系统的所有细节，那就不可能再编码了。而且管理这些部件(硬件)，并加以优化使用，是一件挑战性极强的工作。

操作系统，它的任务是为用户提供一个更好、更简单、更清晰的计算模型，并管理计算机的各种设备。

GUI和Shell实际上并不算是操作系统的一部分，尽管这些程序使用操作系统来完成工作。

多数计算机拥有两种运行模式：内核态和用户态 软件中最基础的部分是操作系统，它运行在内核态。在这个模式中操作系统具有对所有硬件的完全访问权，可以执行机器能够运行的任何指令。 软件的其余部分运行在用户态下，用户态只使用了机器指令中的一个子集。在用户态中无法使用会影响机器运行和进行IO操作的指令。（这些指令通过操作系统调用接口来执行） 用户接口程序（shell或gui）是用户态程序的最低层次，大量使用操作系统接口 （不过在嵌入式开发领域，这些区别比较模糊，应为其开发的东西更接近底层）

操作系统和应用程序的差距：操作系统更加大型、复杂和长寿，Linux或者Windows操作系统仅仅是内核源码至少有500万行，印刷成书的话会有100本（1000页，50行一页），加上内核以外的其他基本代码window会有超过7000万行（还不包括office等基本应用程序）

1.1 什么是操作系统#

1.1.1 作为扩展机器的操作系统 自顶向下：操作系统向应用程序提供基本的抽象

抽象是管理复杂性的一个关键。好的抽象可以把一个几乎不可能管理的任务划分成两个可管理的部分。其第一部分是有关抽象的定义和实现，第二部分是随时用这些抽象解决问题。

操作系统的任务是创建好的抽象，并实现和管理它所创建的抽象对象。

即使小心翼翼怀着对Macintosh机器的工业设计师的尊敬，还是不得不说，硬件是丑陋的。真实的处理器、内存条、磁盘和其他装置都是非常复杂的，很多硬件的接口都是困难、可怕、特殊和不一致的。有时是由于需要兼容旧的硬件、有时是为了节省成本，也有时硬件设计者并没有意识到他们给软件设计者带来了多大的麻烦(...hhhhh)

操作系统的另一个任务是，隐藏硬件，呈现给程序（以及程序员）良好、清晰、优雅、一致的抽象。（操作系统将丑陋转变为美丽）

1.1.2 作为资源管理者的操作系统 自底向上：操作系统管理一个复杂系统的各个部分

1.3 计算机硬件简介#

讨论计算机硬件和计算机组织的书籍《Tanenbaum》和《Austin》《Patterson》《Hennessy》 cpu、内存以及I/O设备都由一条系统总线连接起来并通过总线与其他设备通信。 现代个人计算机更加复杂，包含多重总线。目前这一模式还是够用的。

1.3.1 处理器CPU 计算机的大脑是CPU，它从内存中取出指令并执行。 在每个CPU周期中，首先从内存中取出指令，解码以确定其类型和操作数，接着执行，然后取指、解码并执行下一条指令。按照这一方式，程序被执行。
